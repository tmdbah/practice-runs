## Goals

<!-- Goals & requirements -->

Add visual indicators to the availability grid so a proposed Session (and player RSVPs) are visible at a glance without scrolling to the Sessions list:
- Day-level: the grid's column header for a date with a proposed session gets a gold accent (pill treatment, reusing the player-number badge styling).
- Player-level: a player's cell gets a small gold checkmark badge (bottom-right corner) when they've RSVP'd "in" (`Rsvp.status === "ANYTIME"`) for a session on that date.
- Both markers must coexist without collision when a cell already shows the existing This-Week-override/has-note dot (top-right corner) — different corners, both visible.
- Applies identically in Usual and This Week modes; UI-only, no schema changes.

See full plan: `/Users/tmdbah/.claude/plans/refactored-zooming-swing.md`

## Notes

<!-- Any extra notes -->

`sessions` state is lifted from `SessionsView` up into `TeamGrid` (single owner, passed to both `AvailabilityGrid` and `SessionsView`) so grid indicators update live after propose/RSVP/delete, matching the app's existing optimistic-UI conventions. `page.tsx` is untouched. New pure helpers (`groupSessionsByDate`, `hasSessionOnDate`, `isPlayerRsvpdIn`) live in `AvailabilityGrid.tsx` itself (not `src/lib/`), matching the `TeamWindowCard.tsx`/`computeDefaultIndex` precedent — single call site, no Prisma, per the architecture doc's own `src/lib/` promotion criterion (reuse across Server Components/API routes).
